# Jira Interaction

Use this skill when working with Jira issues.

Rules:

1. Confirm the issue key and project before changing state.
2. Use `{{JIRA_PROJECT_KEY}}` for reusable templates.
3. Prefer comments that summarize evidence, current status, and next action.
4. Only transition issues when the workflow state matches the actual work state.
5. Avoid pasting private logs; upload artifacts and link to them when appropriate.
